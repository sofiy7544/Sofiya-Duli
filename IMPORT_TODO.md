> Файл-задание для следующей сессии Claude Code. Появился потому, что сетевая
> политика окружения блокировала `toytoytrade.com` (egress-proxy → 403).
> Выполнять **только** в сессии, где сеть разрешена (Unrestricted / allowlist с
> `toytoytrade.com`).

## Цель
Перенести **100% ассортимента** сайта https://toytoytrade.com/ru в каталог сайта
DULI TOYS (`index.html`, массив `PRODUCTS` / `LINES`). Полностью, без пропусков.

## Требования заказчика (дословно)
- Перенести 100% ассортимента, без исключений — все категории и все товары.
- Создать карточку на каждый товар.
- Перенести все доступные поля из карточки: **название, артикул, бренд,
  описание, характеристики, категории** и любые другие поля, если есть.
- **Фотографии НЕ загружать и НЕ переносить.**
- Если товар уже существует — **обновить**, а не создавать дубликат
  (ключ дедупликации — артикул; если артикула нет, то нормализованное название).
- Ничего не выдумывать, названия товаров не менять.
- Работать до полного соответствия каталога ассортименту сайта.

## План выполнения
1. **Проверить доступ:** `curl -sS "$HTTPS_PROXY/__agentproxy/status"` — убедиться,
   что нет `connect_rejected` для `toytoytrade.com`. Затем тестовый WebFetch/curl.
2. **Собрать структуру:** получить sitemap (`/sitemap.xml`, возможны вложенные
   sitemap'ы товаров/категорий) ИЛИ обойти меню категорий. Определить движок
   (похоже на Хорошоп/Prom/OpenCart — проверить наличие `export`, YML-фида
   `/export/` или `price.xml`, это ускорит сбор).
3. **Обойти все категории с пагинацией** — собрать полный список URL товаров.
   Логировать количество по каждой категории, чтобы ничего не потерять.
4. **По каждому товару** извлечь: название, артикул (SKU), бренд, описание,
   характеристики (все пары ключ-значение), категорию/подкатегорию, цену,
   наличие, штрихкод/EAN — всё, что есть. **Изображения игнорировать.**
5. **Сформировать данные каталога** — заменить/дополнить массив товаров в
   `index.html`. Учесть, что текущий каталог использует поля:
   `art, cat, line, name, sub, age, mat, size, pack, moq, opt, retail, stock`.
   Добавить недостающие поля (brand, description, characteristics, category path)
   и при необходимости расширить карточку/таблицу и модалку под новые данные.
6. **Дедупликация** по артикулу: существующий товар обновляем, новый добавляем.
7. **Проверка полноты:** число товаров в каталоге == число товаров на сайте
   (сверить по счётчикам категорий). Прогнать рендер через headless Chromium.
8. **Публикация:** commit + push в `main` (GitHub Pages авто-деплой);
   отдать архив пользователю; кратко отчитаться, сколько категорий/товаров.
